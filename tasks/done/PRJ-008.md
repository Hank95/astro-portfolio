# Fix resume PDF download to exclude site chrome

## Objective
The "Download Resume" button calls `window.print()`, which renders the full page including the navigation bar, footer, and excess padding. The print CSS in `resume.astro` attempts to hide these elements but the result is still a messy PDF with site chrome. The downloaded resume should look like a clean, standalone document.

## Acceptance Criteria
- [x] Downloaded PDF shows only the resume content — no navbar, no footer, no site padding
- [x] PDF fits cleanly on standard letter-size pages
- [x] PDF typography and spacing look professional (not a screenshot of a webpage)
- [x] Dark mode does not bleed into the printed output
- [x] Button still works as a one-click action (no multi-step download flow)

## Notes
Current approach: `window.print()` with `@media print` CSS in `resume.astro`.
Print CSS hides `header:not(.resume-header)`, `footer`, `nav`, `[class*="fixed"]`, and `.no-print`.
The site Header component is a `<header>` with `class="fixed"` — the selectors should match, but browser print rendering is inconsistent.
Consider: tightening print CSS (remove `pt-24` padding, force white bg on body), or generating a static PDF at build time.
